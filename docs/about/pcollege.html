College Projects

Last modified: Sat Mar 17 2018 16:45:16 GMT+0800 (Malay Peninsula Standard Time)

College Projects

Below are a list of individual and group projects that I did in college. Some of the projects are open source and available in my Github.

Individual Projects

Ruby on Rails Tutorial | Introduction To Web Development - Sept 2016 - Dec 2017

  • A Ruby on Rails Tutorial and Web Development that I wrote while working for two professors that teaches Software Engineering class in San Jose State University Computer Engineering Department, CMPE131.

Ruby SNMP Manager - Oct 2017 - Nov 2017

  • Wrote a Ruby SNMP Manager script based on SNMP V2. Features include Get system information, List all interface, List all neighbor, Plot traffic, and Plot each interface's traffic.

Event Jekyll Theme - June 2017 - Aug 2017

  • Created a theme pacakge based on Jekyll on Rails to be used for conference and event purposes. Available to everyone for free and completely open source.

Data Analytics Web Application - June 2017

  • This is a sample web application written in Ruby on Rails to analyze the sample data for CMPE180-38/CMPE226 Database Systems Workshop. The entire web application took approximately 36 hours.
  • The datasets were downloaded from TransparentCalifornia.com and imported as CSV to Ruby on Rails Web Application. In each year, there are approximately 70000-90000 rows of data. The dataset are too large and there was nowhere the data can be hosted for free in the cloud. However, the web application itself is hosted in Heroku PaaS.
  • The web application allows the user to choose a year and turns the data in the year to a more readable form. Sample operations that user can carry out include obtaining mean based on job title, filtering column based on the input range, sorting data based on a column in ascending or descending order, plotting the data into bar chart or pie chart, etc.
  • Note: 2016 (3K), Year 2016 with 3000 rows of data, as the data source will be chosen by default when the application environment is in production mode. This is due to the amount of data in each year is too big (>70000 rows) per year. Using the huge data will cause the server to run out of memory and crash.

Calculator - May 2017

  • Designed a calculator using system level architecture, RTL, that is capable to perform add, subtract, multiply, and divide operation.
  • Used Verilog to create control unit and data path for the modules.Performed functional verification of the data path, control unit, and integrated system using self-checking test bench along with ASM and FSM charts.
  • Performed hardware verification using Xilinx Nexys 4 DDR Artix-7 FLGA board.

Data Logger - April 2017

  • Designed and built a data logger device using LPC1769 board and a SPI flash module.
  • Data logger were written in C and capable of reading device ID, wiping flash module, reading from and storing to flash module.

Group Projects

Final Year Project: IIoT - Smart Parking System - Dec 2016 - Dec 2017

  • Developed a system (hardware and software) to provide a guarantee parking spot for garage user.
  • System includes microcontrollers with cameras, image recognition API to analyze license plates, Rails Web App with reserve and pay for a parking spot online or in garage, scalable Rails API App for REST API calls from microcontroller and Web App, statistics of parking garages, real time notification for garage administrator when a license plate is mismatched, and E-mail notification for users and administrators.

RTOS RGB MP3 Player with Game - Nov 2017 – Dec 2017

  • Two weeks long academic project that allows students to use their creativity to create MP3 Player. RTOS is used as the operating system for the development board.
  • Tasks completed including writing device drivers for RGB LCD (I2C), MP3 Decoder (SPI), and power circuit.
  • The source code is written in C++.
  • Features implemented include play, pause, stop, next, previous, fast forward, bass treble, volume control, and game (Truck Lane).

CorgiBot - 24 hours Hackathon Project - May 2017

  • A completely wireless and battery powered robot that is capable of searching for an item in your house (or warehouse).
  • The robot was built based on the materials (cardboard) we found at the Hackathon and the hardware we brought to the 24 hours AngelHack San Francisco Hackathon.
  • Technology used includes Amazon AWS, Redis, Heroku, PostgreSQL, Intel Edison, SparkPost, Amazon S3, Python, Amazon AWS Lambda, Ruby on Rails, Alexa, Action Cable, and Amazon AWS Rekognition.
  • Awarded Top 10 teams in AngelHack Global Hackathon Series: San Francisco & Best use of Amazon Web Services award

NoNameCompiler - April 2017 - May 2017

  • Invented a new programming language and defined a grammar for it.
  • Used JavaCC compiler-compiler to generate a compiler for the team’s language. Compiled programs into the assembly language of the Java Virtual Machine (JVM).
  • Assembled the assembly code into byte code for the JVM (create .class files).
  • Ran programs written in the team’s programming language on the JVM.

SOMII - Interactive Educational Stand - Oct 2016 - Sep 2017

  • An interactive educational booth can that is able to recognize and give the user the feedback about the item the user put on the platform.
  • The image is taken from Intel Edison microcontroller and processes on Amazon AWS IoT using Amazon Rekognition (Image Detection and Recognition API).
  • The entire booth is completely wireless and runs on rechargeable battery.
  • This project was awarded 4th Place in Paseo Public Prototyping Challenge (2016-2017).

Shortest Path Finder - Sep 2016

  • A Website that is powered by Ruby on Rails that allows users to input two valid points from the drop down list and the system will generate the shortest path to travel from two points on foot.
  • All the points are loaded by our team members and shortest path are generated by Dijkstra Algorithm.
  • The shortest path will then be displayed on Google Maps using Google Maps API.

Cloud Translate Platform - Mar 2016 - May 2016

  • A web application focused in Database system. Our application a proof of concept for our Database class.
  • PHP is used as front end and MySQL is used as backend database engine.
  • Web Application is hosted by Heroku and Database is hosted by ClearDB.
  • The website is created without any help of any framework Since this is a database class, we chose to query manually and use basic front end. 3NF is achieved and there is no transaction.

Campus Study Spot Finder - Jan 2016 - May 2016

  • A web application created using Ruby on Rails web application framework for Software Engineering class (CS160/CMPE131/SE131).
  • The purpose of this web application is to allow students in San Jose State Univeristy to create and rate locations, leave and upvote reviews on the study spot that they prefer.
  • Postgres is used as our database engine and up to 4 tables are joined to get a specific result.
  • Sendgrid is used as our Mail Server.
  • Bcrypt is used to hash/encrypt user password and reCAPTCHA is used to avoid any spam.
  • Regular expressions are used to make sure correct email is typed in and full name contains first and last name.
  • Used Ruby's activerecord for query
  • Some of the query are done manually due to different attribute name caused by GEM.
  • Memcache is setup to speed up the website and logs are logged background to keep track of the application.

Robot Project - Apr 2014 - May 2014

  • Built, designed and test an autonomous robot that able to go to the beacon and turn off the switch that send the signal and move another beacon from the race arena.
  • Intelitek EasyC V4 Cortex robotic kit is used. Our team's robot completed the task in the least time.

Wind Turbine Project - Mar 2014 - Mar 2014

  • Design, build and test wind turbine blade. Solidworks was used to design the wind turbine blade.
  • Conducted a few test to find the efficiency of the blade and maximum power output.
